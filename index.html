Visual Regression Testing

  • Code that takes and compares screenshots
  • Highlights changes between versions of CSS

PhantomCSS

  • Changing Inherited Projects
  • Building a Project to Pass Off
  • Rapidly Changing UIs
  • Changes when the CSS changes
  • Run as part of Continual Integration
  • Developers: confidence to make visual tweaks
  • Designers: an easier time in visual QA

A/B Testing

  • Build multiple versions of something
  • Randomly show versions to users
  • Analyze the results

Optimizely

  • Highly Transactional Systems
  • Anything Consumer Facing (conversions)
  • In production after Qual Research
  • Do it. Do it live.
  • Developers: It's more data - you're already on board
  • Designers: One less reason to get shot down
